2. Basic COCOMO (Organic) — Formula & Calculation

We use the Basic COCOMO model (Organic mode) with standard coefficients:

  • Effort (Person-Months) = a * (KLOC)b   where a = 2.4, b = 1.05
  • Development Time (Months) = c * (Effort)d   where c = 2.5, d = 0.38

Step-by-step arithmetic (digit-by-digit explanation)

Given:
  KLOC = 3.0
  a = 2.4
  b = 1.05
  c = 2.5
  d = 0.38

1) Compute (KLOC)^b = 3.0^1.05
   Break into 3.0^(1.00) * 3.0^(0.05)
   3.0^1.00 = 3.0
   3.0^0.05 ≈ e^(0.05 * ln(3.0))
   ln(3.0) ≈ 1.0986122886681098
   0.05 * ln(3.0) ≈ 0.05493061443340549
   e^(0.05493061443340549) ≈ 1.056467308549538
   Therefore 3.0^1.05 ≈ 3.0 * 1.056467308549538 ≈ 3.169401925648614

2) Effort = a * (KLOC)^b
   Effort ≈ 2.4 * 3.169401925648614
   Effort ≈ 7.606564621556673  (person-months)
   Rounded for reporting: Effort ≈ 7.61 person-months

3) Schedule (Months) = c * (Effort)^d
   First compute (Effort)^d = 7.606564621556673^0.38
   ln(7.606564621556673) ≈ 2.0281921462793356
   0.38 * ln(7.606564621556673) ≈ 0.7713129952143735
   e^(0.7713129952143735) ≈ 2.1619799024787734
   Therefore (Effort)^d ≈ 2.1619799024787734
   Schedule ≈ 2.5 * 2.1619799024787734 ≈ 5.404949756196933 months
   Rounded for reporting: Schedule ≈ 5.40 months
Summary (rounded):
  • Estimated code size: 3,000 LOC (3.0 KLOC)

Interpretation / Practical note

COCOMO basic gives a rough engineering estimate. For student projects that use frameworks (React, Django, etc.), generated code, and libraries, the actual coding effort is often lower than raw COCOMO estimates because frameworks provide large amounts of reusable functionality. In your handwritten justification you should:

  1. Mention that MODP/TOOL/REUSE factors (use of frameworks, libraries, templates) reduce actual effort.
  2. Explain schedule pressure (SCED) if the semester deadline is tight.
  3. State that COCOMO provides a baseline and that remaining discrepancy is justified by reuse and higher-level components.
